# 当前轮式机器人环境感知与避障调研

更新：2026-07-16

## 结论先行

当前可部署的轮式机器人避障不是一个独立的“避障算法”，而是一条分层闭环：**多传感器测量 -> 时空环境表示 -> 全局路径/局部轨迹 -> 底盘或全身控制 -> 独立安全制动**。其中全局层负责绕开已知静态结构，局部层对新出现的障碍物和人做重规划，最后的安全层必须能够绕过普通规划器直接限速或停车。

对于带上身的轮式人形/移动操作机器人，二维底盘 footprint 已经不够：机械臂、躯干、携带物和相机本身都会造成碰撞或遮挡。因此较先进的系统把环境表示为三维体素/ESDF，并把底盘位姿与上身关节共同作为优化变量；底盘可以让位、转向或后退，手臂则在同一次求解中回缩、绕行并保持末端任务。

## 1. 系统如何分层

```text
轮编码器 + IMU + GNSS/UWB(可选) -------\
2D/3D LiDAR + RGB-D/双目 + RGB ---------> 时间同步、外参标定、状态估计
短距 ToF/超声/触边/安全激光 ------------/             |
                                                  位姿、点云、语义、目标轨迹
                                                          |
        离线静态地图 <--- SLAM/建图 ---> 局部滚动占据图/体素图/ESDF
                                                          |
       全局规划(A*/Hybrid-A*/Smac)        局部规划(DWB/TEB/MPPI/MPC)
                                                          |
           底盘速度控制                 全身 QP/CBF/MPC (底盘 + 躯干 + 双臂)
                                                          |
                     安全激光/碰撞监视器 -> 限速、制动、急停
```

这里的关键工程原则是：**规划层可以失败或超时，安全层不能依赖规划层**。非安全认证的软件碰撞监视器有价值，但不能替代安全级 LiDAR、控制器和急停回路。

### 2.1 定位与建图

典型室内轮式平台采用“轮速 + IMU 提供高频预测，LiDAR SLAM 或视觉/视觉惯性 SLAM 提供地图约束”的组合。输出不是单一地图，而至少包括：

1. **全局静态地图**：通常是二维 occupancy grid，供跨房间路径规划和禁行区管理。
2. **局部滚动地图**：机器人周围数米至十余米的实时障碍层；旧观测要被射线清除或按时间衰减，防止移动的人留下“鬼影”。
3. **三维地图/距离场**：对移动操作臂和轮式人形尤其必要。体素占据图可描述不同高度障碍；ESDF（Euclidean Signed Distance Field）进一步提供距离及梯度，供轨迹优化直接计算避障代价/约束。

开源的 [RTAB-Map ROS](https://github.com/introlab/rtabmap_ros) 支持 RGB-D、双目和 3D LiDAR 的 ROS 2 接入及 Nav2 示例；[Isaac ROS Nvblox](https://github.com/NVIDIA-ISAAC-ROS/isaac_ros_nvblox) 利用深度与位姿实时重建三维场景，并输出 Nav2 可用二维代价地图。两者都适合作为原型，但计算预算、传感器时间同步和外参漂移必须在真机上单独验证。

### 2.2 从点云到可避障的时空表示

普通二维 costmap 将每个栅格标为可通行/障碍/未知，适合纯底盘；带上身的平台更常采用以下组合：

- **静态占据 + 膨胀层**：把机器人半径、定位误差和制动距离折算为代价，供全局路线绕开固定物。
- **滚动三维体素层**：将 LiDAR/RGB-D 点云写入局部体素，基于传感器视锥射线清除自由空间，并让未再次观测到的体素随时间衰减。2020 年的 Spatio-Temporal Voxel Layer（STVL）是代表实现，基于 OpenVDB 维护稀疏三维体素并支持衰减；[论文/代码](https://github.com/SteveMacenski/spatio_temporal_voxel_layer) 为 LGPL-2.1。
- **ESDF/SDF**：对任意机器人球、胶囊体或网格查询“到障碍的最近距离”和梯度。它使全身轨迹优化避免在每步都做大量离散碰撞检测。
- **目标轨迹与预测占用**：检测与跟踪模块估计行人/车辆的位置、速度、类别和不确定性；局部 MPC/速度障碍方法在预测时域内对其未来位置留出裕量。对人群环境，预测错误时应退化为保守 TTC 限速，而不是相信长时预测。
- **传感器可见性/遮挡**：机械臂举起、负载遮挡、转弯时视野变化会形成“未知空间”。较稳妥的策略是对未知区保守降速、主动改变视点，或把视野保持列进优化目标。

## 3. 避障如何落到运动控制

### 3.1 纯轮式底盘：两级规划 + 安全监督

1. **全局规划**：在静态或慢变地图上找路线。差速底盘可使用 A*/Dijkstra；存在最小转弯半径或倒车约束时使用 Hybrid-A*、lattice 或 Smac；全向底盘可以直接在平面速度空间寻路。
2. **局部规划/控制**：以局部 costmap、当前速度和运动学约束为输入，输出短时速度或轨迹。DWB/DWA 从速度候选中评分，TEB 将轨迹表示为时间弹性带，MPPI 采样控制序列并用代价选择动作，MPC 显式预测状态与约束。
3. **恢复行为**：局部无解时减速、清图、原地转向、后退或请求新全局路线；不能仅持续“挤”向障碍。
4. **独立制动**：以保护区、TTC 或制动距离检测 imminent collision，直接覆盖普通 `cmd_vel`。Nav2 的 [Collision Monitor](https://docs.nav2.org/configuration/packages/collision_monitor/configuring-collision-monitor-node.html) 提供 stop、slowdown、速度上限与 TTC approach 四类模型，但官方明确它是 CPU 软件节点，并非安全认证。

[Nav2](https://nav2.org/) 是当前 ROS 2 轮式机器人最完整的开源导航基础设施：覆盖感知接入、定位、全局/局部规划、行为树、恢复动作和碰撞监视。它并不等于“已解决感知”，其输入质量、传感器覆盖和速度约束决定最终可靠性。官方也提供 [纯双目视觉导航示例](https://docs.nav2.org/tutorials/docs/using_isaac_perceptor.html)，用立体相机和 IMU 完成定位、建图与避障；对工业/人机共存场景，仍建议 LiDAR 或安全传感器冗余。

### 3.2 移动操作臂/轮式人形：把全身放进避障变量

设底盘状态为 `q_b = [x, y, yaw]`，上身关节为 `q_u`，全身状态为 `q = [q_b, q_u]`。末端任务通常是跟踪 `x_ee(q)`，避障不能只限制底盘速度，而需同时求解：

```text
min  ||J_ee(q) qdot - xdot_ee_ref||_W
   + ||qdot - qdot_nominal||_R
   + collision / visibility / smoothness cost

s.t.  base kinematics, joint/velocity/acceleration limits,
      d_i(q, map) >= safety_margin_i,
      optional: object pose, force, camera visibility constraints
```

常见落地方式：

- **优先级 QP / Whole-Body Control (WBC)**：末端任务、姿态、关节限位和避障速度约束在一个二次规划中求解。计算快，适合 100 Hz 左右的反应层；冗余度决定是底盘让位还是手臂回缩。
- **CBF-QP（控制屏障函数）**：把安全距离转成对 `qdot` 的不等式约束。优点是有明确的连续时间安全形式；难点是动态障碍、传感噪声和“势阱/伪平衡”会使机器人停住，需要任务重规划或附加方向性约束。
- **全身 MPC/NMPC**：在数百毫秒到数秒时域内共同优化底盘和上身轨迹，能纳入速度、加速度、力、稳定性、末端精度及动态障碍预测。更适合高速运动和“边走边操作”，但需要良好的距离场、状态估计和可控的求解时延。
- **先全局再局部的混合式**：全局采样/搜索先找拓扑可行通路；局部 WBC/MPC 用最新感知做安全修正。这是目前兼顾复杂空间和实时反应最常见的路线。

全身规划的碰撞几何常把底盘、躯干、双臂和载荷近似为球/胶囊集合。它比精细网格保守，但能快速对 ESDF 查询距离。对于末端高精度任务，应将**末端位姿与接触/力任务设为高优先级**，把底盘移动和冗余关节调整作为补偿量；当安全约束无法满足时，必须允许任务速度降低或暂停，不能强行维持末端轨迹。

### WBC 与动态避障的阅读顺序

若目标是实现“底盘和上身统一解算、对快速动态障碍有反应、末端仍保持任务精度”，建议按以下顺序阅读：

1. **SEWB (2026)**：最直接的 CBF-QP 动态全身安全滤波；重点看 ACI 如何使用障碍速度解除 CBF 的伪平衡。
2. **Adaptive WBC (2025)**：最接近人机共存工厂的工程流程；重点看分区降维、相对距离速度约束和全身 QP 目标构造。
3. **RAMPAGE (2024)**：补齐三维感知、ESDF 与长时域全身轨迹优化；重点看球分解碰撞模型及搜索-优化两阶段接口。
4. **Perceptive MPC (2020)**：补齐末端视觉避障、触觉交互力和连续移动操作；适合定义高精度操作时的任务约束。

这些工作并不等价：CBF-QP/WBC 更适合高频反射层，ESDF + MPC/DDP 更适合前瞻轨迹层。实际系统通常两者并用，并由独立制动层兜底。

| [Robot Navigation in Crowded Environments Using Deep Reinforcement Learning](https://doi.org/10.1109/IROS45743.2020.9341540) | IROS 2020 | 面向真实移动机器人在有限视场、静态障碍与人群混合环境中导航 | 静态物和动态人群分别编码，先模仿学习再 DRL；论文报告真机实时运行。适合研究“感知受限 + 人群”的学习局部策略，但要加安全过滤器再部署 | 论文项目提供视频；未确认可复现实装官方代码 |
| [DRL-VO: Learning to Navigate Through Crowded Dynamic Scenes Using Velocity Obstacles](https://arxiv.org/abs/2301.06512) | T-RO 2023 | 将经典 VO 的安全先验与 LiDAR 端到端局部策略结合，且有多平台实机验证 | 输入短时 LiDAR 历史、附近行人运动学状态和子目标；RL reward 含 VO 项，输出前进速度与转向。论文报告无需再训练即可迁移到不同真实人群和另一底盘平台 | [项目/代码](https://github.com/TempleRAIL/drl_vo_nav)，以仓库当前许可证为准 |

**推荐阅读顺序（纯轮式动态避障）**：先以 `DWA -> TEB -> VO/ORCA` 建立速度空间和时空轨迹优化基础；随后读 `Dong 2021` 理解 2D LiDAR 感知跟踪的完整链路，读 `TAMP 2020` 处理预测不准，最后读 `DRL-VO 2023` 评估学习策略是否能替换或辅助传统局部规划器。对工程系统而言，DRL/社会导航策略只能是局部决策层，仍需由碰撞监视器或 CBF/TTC 安全层做最终约束。

## 5. 针对不同机器人形态的选型

| 形态/目标 | 推荐感知和地图 | 推荐避障控制 | 关键注意点 |
| --- | --- | --- | --- |
| 低速室内 AMR | 2D LiDAR + 编码器/IMU；二维静态图 + 滚动 costmap | Nav2 全局规划 + DWB/MPPI + TTC/保护区监视 | 传感器高度要覆盖桌腿、货叉等；安全区按最大速度和制动能力标定 |
| 高速轮式配送/巡检 | 3D LiDAR 与相机互补；局部体素/ESDF + 动态目标跟踪 | 预测式 MPPI/MPC，独立安全 LiDAR 制动 | 速度越高越不能只靠 costmap；延迟、定位误差和制动距离要一起进入安全裕量 |
| 单臂移动操作 | 头部/腕部 RGB-D + LiDAR；局部三维体素/ESDF | 全局路径 + 全身 QP/CBF 或 MPC | 将机械臂、托盘、抓取物写进碰撞模型；操作时让底盘、臂和躯干共同让位 |
| 轮式人形、双臂高精度操作 | 360 度远距 LiDAR、头/胸 RGB-D、腕部近距深度、冗余短距安全传感 | 任务优先级 WBC 作高频层，NMPC/轨迹优化作预测层，安全控制器独立 | 重点处理自遮挡、双臂互碰、负载外形、相机视野和末端抖动；不要把感知延迟直接送入高速关节环 |

## 6. 一套可实施的参考架构

面向“轮式底盘 + 上身操作 + 末端高精度”的项目，建议先采用以下可替换架构，而非从端到端避障网络开始：

1. **硬件覆盖**：车体四周 3D LiDAR 或 2D 安全 LiDAR，头/胸 RGB-D 覆盖上半身工作区，腕部深度相机补充抓取附近盲区，编码器 + IMU 提供高频本体状态。为低矮、透明或反射风险物加短距 ToF/保险杠冗余。
2. **状态与地图**：`robot_localization` 类滤波器融合轮速/IMU；LiDAR/视觉 SLAM 提供全局校正。维护全局静态二维图、局部时空三维体素图和局部 ESDF；动态物保留速度、类别、协方差与观测时间。
3. **慢层（约 1-5 Hz）**：任务规划、语义禁区和全局/拓扑路线；输出底盘走廊、操作关键帧与末端目标，而不是逐关节命令。
4. **中层（约 10-30 Hz）**：更新局部体素/ESDF，预测动态障碍，运行全身轨迹优化或 MPC；输出参考 `q, qdot` 与底盘速度。
5. **快层（约 100 Hz 或按硬件能力）**：WBC/QP 跟踪末端和姿态任务，实时约束全身距离、关节限制和速度限制。这里应只使用时间对齐后的局部几何，不等待云端或大模型推理。
6. **安全层（独立、尽可能更高频）**：基于安全传感器的分级保护区、TTC、硬件急停和速度安全上限；它有权覆盖上层命令。把载荷、伸出的手臂和当前速度映射到动态保护区。

### 接口建议

感知到控制的消息不应只传“最近障碍距离”，至少包含：

```text
timestamp, frame_id, robot pose covariance,
local occupancy/ESDF version and valid horizon,
obstacle track {id, shape, pose, velocity, covariance, class, last_seen},
sensor visibility / unknown-space mask,
robot self-model + payload geometry version,
safety envelope {speed limit, stop zone state, reason}
```

控制器应回传预测轨迹、最小距离、约束激活状态、求解耗时和降级状态，便于记录“看见了但没有停”“因遮挡误停”等问题。所有数据需要统一时钟；以过期点云做高速避障是比算法选择更常见的失效原因。

## 7. 工程风险与验证清单

- **盲区**：低矮障碍、悬空障碍、传感器被手臂/负载遮挡、倒车方向。必须用实体障碍做覆盖测试，不能只看 RViz 点云。
- **动态物体**：对人和叉车使用速度预测，但按不确定性扩大安全区；跟踪丢失时转为 TTC 保守限速。
- **定位误差**：将定位协方差、地图分辨率、外参误差和制动距离共同计入膨胀半径/CBF margin，而不是只留固定 5 cm。
- **全身模型遗漏**：夹爪、线缆、托盘和抓取物必须进入自碰/环境碰撞几何；负载变化应触发模型版本更新。
- **求解超时**：定义明确退化策略：保持上次已验证轨迹并减速、切换局部 QP、或停车；禁止继续执行过期轨迹。
- **安全认证边界**：软件 costmap、深度相机、普通 LiDAR 和 AI 检测都不天然满足功能安全。人员密集/高动能场景应由安全级传感器、独立安全控制器和风险评估闭环承担最终安全职责。
