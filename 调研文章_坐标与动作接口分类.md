# 调研文章：坐标系与动作接口分类

调研日期：2026-07-17。

## 1. 分类口径

这里的分类回答的是**上层策略/规划器传给下层控制器的主要任务接口**，而不是求解器内部一定会出现的状态变量。比如，一个 WBC 即使内部优化了底盘位姿和关节角，只要上层给它的是末端位姿，它仍归入末端接口；不能据此归到“全身关节端到端”。

| 编号 | 类型 | 含义 |
|---|---|---|
| 1 | 末端世界坐标系 | 给出世界/任务坐标系中的绝对末端位姿或轨迹。 |
| 2 | 末端目标相对坐标系 | 给出相对机器人基座、当前末端或任务物体/关键点的目标末端位姿。 |
| 3 | 末端偏移量 | 给出当前末端的增量（delta pose/twist）；坐标系必须以原文为准。 |
| 4 | 关节角目标 + 底盘偏移量 | 策略显式分别输出底盘与关节/上肢目标。 |
| 5 | 全身关节端到端 | 策略直接输出全身关节位置、速度或力矩，不经末端任务空间接口。 |
| 0 | 不适用/混合 | 不以这五类末端-全身接口为主，例如纯底盘导航、安全过滤或论文未把接口说清。 |

“主类型”只选最能代表部署接口的一类；“补充类型”用于确实存在两种动作模式的论文。本文中的“文中对应描述”均是可定位的原文短句或公式性释义，避免由标题推断坐标系。

## 3. 逐篇依据

### 1. Keep it Upright: Model Predictive Control for Nonprehensile Object Transportation with Obstacle Avoidance on a Mobile Manipulator

- **任务**：端着托盘在静态/动态障碍物间快速运输未抓取物体，同时防滑、防倾倒；实机使用 Ridgeback + UR10。
- **分类：1，末端世界坐标系（任务空间绝对目标）**。目标代价是末端位置到目标位置的距离，文中没有把策略动作定义为 EE delta 或直接关节动作；MPC 的状态则包含底盘姿态和臂关节，并在内部联合求解。
- **文中对应描述**：原文称其“optimizes the joint-space trajectory online directly from task-space objectives and constraints”，并在代价项中明确“不 include a desired orientation”；即位置目标在任务/全局空间，托盘姿态由平衡约束决定。
- **注意**：这不是类型 5。它的优化变量虽是关节空间轨迹，但对上层任务的表达仍是任务空间目标。
- **来源**：[论文 PDF](https://www.dynsyslab.org/wp-content/papercite-data/pdf/heins-ral23.pdf)，[代码 upright](https://github.com/utiasDSL/upright)，RA-L 2023，DOI: [10.1109/LRA.2023.3324520](https://doi.org/10.1109/LRA.2023.3324520)。

### 2. Interactive Navigation with Adaptive Non-prehensile Mobile Manipulation

- **任务**：在 NAMO 中推动可移动室内障碍；对象质量、摩擦等动力学未知。
- **分类：0，不适用**。论文的核心状态/模型是物体的 learned **SE(2)** dynamics，并将其接入 MPPI；Shmoobot 是平衡轮式底盘，不存在机械臂末端、关节角 WBC 接口。
- **文中对应描述**：摘要写明 “learned SE(2) dynamics representations” 和 “integrated into Model Predictive Path Integral (MPPI) control”。这里的 SE(2) 是平面对象/机器人运动状态，不应误归入“末端目标相对坐标系”。
- **来源**：[arXiv](https://arxiv.org/abs/2410.13418)，[项目页](https://cmushmoobot.github.io/AdaptivePushing/)，RA-L 2026，DOI: [10.1109/LRA.2025.3636004](https://doi.org/10.1109/LRA.2025.3636004)。

### 3. A Holistic Approach to Reactive Mobile Manipulation

- **任务**：无全局轨迹规划地完成视觉闭环抓取、拾放；底盘和机械臂作为一个整体反应式运动。
- **分类：1，末端世界/任务坐标系目标位姿**。上层行为树提供 desired EE pose，控制器以全身雅可比/QP 将其映射到基座与臂的速度。
- **文中对应描述**：摘要的核心表述是 “achieve a desired end-effector pose”，且明确 “treat the arm and base degrees of freedom as a holistic structure”。这支持“EE pose 为接口、底盘+臂为协同执行变量”。
- **来源**：[arXiv](https://arxiv.org/abs/2109.04749)，[项目/代码](https://jhavl.github.io/holistic/)，RA-L 2022，DOI: [10.1109/LRA.2022.3146554](https://doi.org/10.1109/LRA.2022.3146554)。

### 4. Real-time Whole-body Motion Planning for Mobile Manipulators Using Environment-adaptive Search and Spatial-temporal Optimization (REMANI-Planner)

- **任务**：复杂静态环境中实时产生安全、敏捷、动力学可行的移动机械臂全身轨迹并完成抓取类任务。
- **分类：1，末端世界/任务坐标系目标**。环境自适应搜索给全身路径初值，时空优化再通过任务约束保证 EE/操作任务；底盘与关节为优化的联合状态。
- **文中对应描述**：项目页说明“task constraints are applied to ensure that the trajectory can fulfill specific task requirements”，且流程是“path ... as an initial guess”后进行“spatial-temporal trajectory optimization”。它是任务空间目标约束下的全身轨迹规划，不是网络直接回归关节。
- **来源**：[项目页](https://robotics-star.com/REMANI-Planner/)，[代码](https://github.com/SYSU-STAR/REMANI-Planner)，ICRA 2024。

### 5. Whole-body End-Effector Pose Tracking

- **任务**：ANYmal 搭载 6-DoF 机械臂在台阶、坡面等粗糙地形中，以全身 RL 精确跟踪大工作空间的 6-DoF 末端姿态。
- **分类：2，末端目标相对坐标系**。具体是目标与当前 EE 的三组关键点位置差，全部在 **base frame** 表达。
- **文中对应描述**：原文定义 command 为 “the positional difference between the current and the target keypoints of the end-effector in the base frame”；三关键点合为 9 维位置命令。这是分类最明确的一篇。
- **来源**：[arXiv](https://arxiv.org/abs/2409.16048)，[OpenReview](https://openreview.net/forum?id=9LODvwGBtl)，ICRA 2025。

### 6. UMI on Legs: Making Manipulation Policies Mobile with Manipulation-Centric Whole-body Controllers

- **任务**：将手持 UMI 收集的真实操作示范迁移到四足带臂机器人，覆盖抓取、非抓取和动态操作。
- **分类：1，末端世界/任务坐标系轨迹**。高层视觉操作策略输出 task-frame EE trajectory，低层 WBC 在身体扰动/腿部运动下维持其相对环境的精度。
- **文中对应描述**：摘要直接写 “The interface between these two policies is end-effector trajectories in the task frame”；项目页进一步对比 body-frame，说明 task-frame 目标不会随基座漂移。
- **来源**：[论文](https://arxiv.org/abs/2407.10353)，[项目页](https://umi-on-legs.github.io/)，[代码](https://github.com/real-stanford/umi-on-legs)，CoRL 2024。

### 7. Humanoid Manipulation Interface: Humanoid Whole-Body Manipulation from Robot-Free Demonstrations (HuMI)

- **任务**：用便携式、无机器人示范采集人全身动作，学习下蹲、跪姿、投掷、行走与双臂操作等人形全身技能。
- **分类：2，局部身体/末端相对目标为主**。人类全身运动不是原样作为机器人关节动作下发，而是经层级重定向与可行性处理后转为人形可跟踪的参考；因此不能归为类型 5。
- **文中对应描述**：摘要称 “hierarchical learning pipeline that translates human motions into dexterous and feasible humanoid skills”。项目页强调采集的是“rich whole-body motion”，而非机器人原生关节指令。
- **边界**：公开摘要未给出足以复现的单一 6-DoF action frame 公式，故将其标为“局部相对接口为主”的保守分类，而非断言具体某只手坐标系。
- **来源**：[arXiv](https://arxiv.org/abs/2602.06643)，[项目页](https://humanoid-manipulation-interface.github.io/)，2026 预印本。

### 8. HiWET: Hierarchical World-Frame End-Effector Tracking for Long-Horizon Humanoid Loco-Manipulation

- **任务**：人形在运动、冲击和长时序任务中保持高精度 EE 操作，同时维持动态稳定。
- **分类：1，末端世界坐标系**。这篇论文的核心就是将问题从 body-centric command 改写为 world-frame EE tracking；高层在世界系联立考虑末端精度与基座摆位。
- **文中对应描述**：摘要为 “We reformulate the problem as world-frame end-effector tracking”，并说明高层产生 “subgoals that jointly optimize end-effector accuracy and base positioning in the world frame”。
- **来源**：[arXiv](https://arxiv.org/abs/2602.06341)，2026 预印本。

### 9. HoMeR: Learning In-the-Wild Mobile Manipulation via Hybrid Imitation and Whole-Body Control

- **任务**：在真实家庭中开柜、清扫、整理软物等；以 20 条示范学习长程移动与近距离精细操作。
- **分类：混合 1 + 3**。keypose 模式输出绝对 6-DoF EE pose（类型 1）；dense 模式输出相对 6-DoF EE delta（类型 3）。统一 WBC 用 IK/QP 同时求底盘和机械臂速度。
- **文中对应描述**：论文称 “keypose actions (6 DoF absolute end-effector poses)” 用于 long-range，且 “dense actions (i.e., delta end-effector actions)” 用于 fine-grained interaction；状态中明确底盘 `q_base=(x,y,theta)` 与臂关节联合输入。
- **来源**：[论文 PDF](https://homer-manip.github.io/assets/paper.pdf)，[项目页/代码](https://homer-manip.github.io/)，arXiv: [2506.01185](https://arxiv.org/abs/2506.01185)。

### 10. HERO: Learning Humanoid End-Effector Control for Visual Whole-Body Open-Vocabulary Object Grasping

- **任务**：利用开放词汇视觉模型在办公室、咖啡厅等环境中走近并抓取日常物体，也包含开门。
- **分类：2，机器人基座坐标系中的 EE 目标**。跟踪策略接收 “desired end-effector pose in the robot frame”，并输出 G1 的全部 29 DoF motor commands；因此任务接口是类型 2，执行动作层才是全身控制。
- **文中对应描述**：原文写 “Given a desired end-effector pose in the robot frame, the end-effector control policy outputs motor commands for all 29-DOFs”。它不应简单归类为类型 5，因为上游命令仍是 EE pose。
- **来源**：[arXiv](https://arxiv.org/abs/2602.16705)，[项目页](https://hero-humanoid.github.io/)，2026 预印本。

### 11. An Architecture for Reactive Mobile Manipulation On-The-Move

- **任务**：底盘不停下、持续驶向下一个高层目标时，抓取静态或不可预测运动物体并完成放置。
- **分类：1 + 3**。目标、当前基座、下一目标均以环境中的 pose 表示；reacher 和末阶段控制器实际下发的是 EE velocity，属于局部增量/速度实现。
- **文中对应描述**：架构输入为 “current pose of the robot, target, and next target”；reacher “calculat[es] end effector velocity commands”，最终控制器也“computes end-effector velocities”。
- **来源**：[arXiv](https://arxiv.org/abs/2212.06991)，[项目页](https://benburgesslimerick.github.io/ManipulationOnTheMove/)，ICRA 2023，页 1623-1629。

### 12. HoMMI: Learning Whole-Body Mobile Manipulation from Human Demonstrations

- **任务**：通过 UMI 加头戴第一视角采集无机器人示范，实现移动、主动感知、双手协作和长时序操作。
- **分类：2，目标相对坐标系**。其跨具身 hand-eye policy 将观察/动作统一变换到左夹爪坐标系，输出双夹爪位姿、look-at 点和夹爪宽度，再由约束感知 WBC 联动手、头、躯干和基座。
- **文中对应描述**：论文摘要指出 WBC “realizes hand-eye trajectories through coordinated whole-body motion”；项目资料给出 23D hand-eye action，采用 left-gripper frame。头部是 relaxed action，而不是复刻人头的完整 6-DoF 位姿。
- **来源**：[arXiv](https://arxiv.org/abs/2603.03243)，[项目页](https://hommi-robot.github.io/)，[论文 PDF](https://hommi-robot.github.io/files/hommi_paper.pdf)，RSS 2026。

### 13. Topology-Driven Parallel Trajectory Optimization in Dynamic Environments

- **任务**：移动底盘在人群等动态环境中，从不同绕行侧/同伦类中并行生成并优化避障轨迹，滚动选择最低代价可行解。
- **分类：0，不适用**。这是地面导航，不含机械臂、EE 或全身关节接口；规划量是移动机器人状态轨迹。
- **文中对应描述**：摘要明确为 “Ground robots navigating” 和 “global planner ... trajectories in distinct homotopy classes”；局部优化器各自尝试一种 evasive maneuver。
- **来源**：[arXiv](https://arxiv.org/abs/2401.06021)，[项目页/代码说明](https://autonomousrobots.nl/paper_websites/topology-driven-mpc)，T-RO 2025。

### 14. SPARK: Safe Protective and Assistive Robot Kit

- **任务**：为人形自主与遥操作提供可组合的安全控制算法、仿真 benchmark 和实机部署接口；支持外部障碍物接近时的主动安全过滤。
- **分类：0，不适用/框架层**。SPARK 不规定高层必须采用哪一种任务坐标或动作表示，而是对上游 nominal policy 的参考控制进行 SSA/CBF/SSS/PFM 等安全修正。因此它可以保护类型 1--5 的任意上游接口。
- **文中对应描述**：摘要称它是 “modular and composable robot control framework”，并提供 “safe control algorithms”；不是一个以 EE pose 或全身关节动作为唯一输出的策略。
- **来源**：[arXiv](https://arxiv.org/abs/2502.03132)，[项目页](https://intelligent-control-lab.github.io/spark/)，[代码](https://github.com/intelligent-control-lab/spark)，IFAC Symposium on Robotics 2025。

### 15. Articulated Object Interaction in Unknown Scenes with Whole-Body Mobile Manipulation

- **任务**：在未知厨房中操作抽屉、柜门、烤箱等大尺寸铰接物体，并应对动态障碍；轮式移动机械臂验证在仿真，四足带臂验证在实机。
- **分类：2，面向目标物的相对操作状态/轨迹**。object-centric 阶段不直接输出机器人关节，而是以 RGB-D 和物体为中心给出 action-conditional manipulation state sequence；agent-centric 最优控制再把它转成对当前机器人与动态环境可行的全身轨迹。
- **文中对应描述**：摘要明确区分第一阶段 “only focuses on the object to provide an action-conditional sequence of states for manipulation”，第二阶段把 whole-body motion control 作为 “optimal control problem” 做安全跟踪。这正是物体/操作目标相对表示到全身执行的两级接口。
- **来源**：[arXiv](https://arxiv.org/abs/2103.10534)，[项目页](https://www.pair.toronto.edu/articulated-mm/)，IROS 2022，页 1173-1180。

## 4. 对接口设计的直接结论

1. **类型 1 是移动操作中最常见、最具跨具身性的上层接口。** UMI on Legs、HiWET、HoMeR 的绝对 keypose、REMANI 和 Holistic 都将“要在环境中到达哪里”与“底盘/关节怎样配合”分开；底层 WBC/MPC/IK 吸收冗余度。
2. **类型 2 更适合局部精细跟踪或跨具身示范。** Whole-body End-Effector Pose Tracking、HERO、HoMMI 都明确用基座或夹爪相对表示，减少全球定位漂移，并使人机形态差异更容易处理；其代价是长距离任务还需要导航/全局重定位。
3. **类型 3 通常不应单独承担长程移动。** HoMeR 将 delta 限定在精细接触阶段；On-The-Move 将速度命令放在闭环 reach/final phase。这是因为增量动作在视觉闭环下精细，但误差会累积。
4. **类型 5 在这批论文中几乎没有作为“对外任务接口”出现。** HERO 的低层输出虽覆盖 29 DoF，但仍以 robot-frame EE pose 为条件；这类“EE 条件 + 全身动作”的结构与纯端到端关节策略应严格区分。
5. **轮式底盘 + 高精度末端的推荐组合**：全局阶段采用类型 1 的绝对 EE keypose/轨迹，近物体时切换类型 3 的视觉闭环 EE delta；WBC/MPC 内部联合求解底盘与关节。HoMeR 是这一路线最清晰的直接例子，Keep it Upright 则是高动态、约束 MPC 的代表。

## 5. 核查边界

- 论文题单包含足式、人形和纯导航工作，本文保留它们是为了完整分类，但已在表中显式标注平台与“不适用”。
- “世界坐标系”在部分传统优化论文中表现为任务空间/地图坐标系的绝对目标，并非每篇都使用 `world` 这个字面名称；凡原文明确写 `base frame`、`robot frame` 或 `left-gripper frame` 的，均优先按原文归入类型 2。
- HuMI 的公开版本没有像 HoMMI/Whole-body End-Effector Pose Tracking 那样完整公开单一步骤 action-frame 公式，故该条采用保守结论；在用于实现选型前，应以作者后续代码或附录为准。

## 6. 维度 A：上层运动如何与底层运动结合

这条维度关注的是“谁决定任务意图，谁处理底盘/躯干/机械臂冗余和约束”。它不同于第 2 节的坐标系分类：同为 EE world pose 接口，底层可以是 MPC、IK-QP 或 RL。

| A 类 | 结合方式 | 对应论文 | 适用性与边界 |
|---|---|---|---|
| A1 | **任务空间目标 -> 联合优化 WBC/MPC/轨迹优化** | 1 Keep it Upright；3 Holistic；4 REMANI；9 HoMeR；11 On-The-Move；15 Articulated Object Interaction | 上层只表达“末端要到哪/完成什么”；下层联合分配底盘与臂。适合高精度末端和大工作空间。 |
| A2 | **高层操作策略 -> 低层学习型全身跟踪器** | 5 Whole-body EE Pose Tracking；6 UMI on Legs；8 HiWET；10 HERO | 高层给 EE task-frame/world-frame/base-frame 命令或子目标，低层 RL 输出全身控制，稳定性与冗余协调被吸收到训练中。 |
| A3 | **人类示范/视觉策略 -> 具身无关轨迹 -> 约束感知 WBC** | 7 HuMI；12 HoMMI | 先处理人到机器人之间的观测/动作形态差异，再由 WBC 变成机器人可行运动；特别适合双手、头部和移动协同。 |
| A4 | **上层导航/交互规划 -> 底盘 MPPI** | 2 Interactive Navigation | 这里只有轮式底盘，尚未出现“底盘与上身统一 WBC”；学习对象 SE(2) 动力学是为 MPPI 的推物决策服务。 |
| A5 | **名义控制器 -> 安全过滤器 -> 机器人** | 14 SPARK | 安全层不替代上层策略和低层控制，而是对其控制量进行约束修正；可叠加在 A1--A4 上。 |
| A6 | **全局同伦引导 -> 并行局部轨迹优化 -> 底盘执行** | 13 Topology-Driven Parallel Trajectory Optimization | 只涉及底盘导航；不含操作策略或全身冗余分配。 |

### A1. 任务空间目标到联合全身优化

- **Keep it Upright**：任务空间目标、平衡和碰撞约束直接进入在线 MPC，底盘和 UR10 关节共同成为求解变量。原文表述为 “optimizes the joint-space trajectory online directly from task-space objectives and constraints”。这是一层式的 task-to-WBC/MPC，而非先规划臂、再移动底盘。[论文](https://www.dynsyslab.org/wp-content/papercite-data/pdf/heins-ral23.pdf)
- **Holistic**：系统把臂和底盘自由度视作 “a holistic structure”；反应式控制器实现 desired EE pose，同时规避关节位置/速度极限并保持可操纵性。上层行为树给任务，速度级 WBC 完成协同分配。[论文](https://arxiv.org/abs/2109.04749)
- **REMANI-Planner**：环境自适应搜索先形成全身路径初值，时空优化再同时考虑 whole-body safety、agility、dynamic feasibility 和 task constraints。其结合点是“全局搜索提供 warm start，局部全身优化兑现任务约束”。[项目页](https://robotics-star.com/REMANI-Planner/)
- **HoMeR**：高层 IL 只学习 EE 动作及模式切换，低层 IK-WBC 把它映射成 base+arm 的联合速度。原文指出 WBC “maps desired end-effector poses to coordinated motion across the mobile base and arm”；keypose 与 dense delta 分别承担长程与精细阶段。[论文 PDF](https://homer-manip.github.io/assets/paper.pdf)
- **On-The-Move**：底盘控制器根据当前目标和下一个目标保持前进；reacher 按“进入可操作范围的剩余时间”生成 EE velocity，最后交给局部任务控制器。因此结合不是一个统一 QP，而是带时间协调的模块化协同。[论文](https://arxiv.org/abs/2212.06991)
- **Articulated Object Interaction**：object-centric 规划器输出与物体动作条件相关的状态序列；agent-centric 规划器将其转为最优控制并安全跟踪。这是典型的“对象层决定接触/操作语义，具身层决定全身运动”。[论文](https://arxiv.org/abs/2103.10534)

### A2. 高层末端目标到学习型 WBC

- **Whole-body End-Effector Pose Tracking**：策略观察本体状态和基座系 EE 关键点误差，在复杂地形上直接输出全身控制；上层命令不是关节序列。原文称其为 “whole-body RL formulation for end-effector pose tracking”。[论文](https://arxiv.org/abs/2409.16048)
- **UMI on Legs**：真实世界操作策略产生 task-frame EE trajectories；仿真训练的 WBC 在 50 Hz 追踪，使腿、身体、臂共同抵消扰动。论文明确称两者的 interface 是 “end-effector trajectories in the task frame”。[论文](https://arxiv.org/abs/2407.10353)
- **HiWET**：高层在世界系生成兼顾末端精度和基座位置的 subgoals，低层策略在稳定性约束下执行。该分层是为解决长时序行走造成的世界系漂移。[论文](https://arxiv.org/abs/2602.06341)
- **HERO**：视觉模块提供目标，IK 将 residual EE targets 变为参考，RL 跟踪器再输出 G1 全 29 DoF 控制。其创新是 learned forward kinematics/base odometry residual、目标修正和重规划的闭环组合。[论文](https://arxiv.org/abs/2602.16705)

### A3--A6. 示范重定向、交互导航、安全层与纯导航

- **HuMI**：便携设备采人类全身运动，再经层次学习把它翻译为 “dexterous and feasible humanoid skills”。这避免了直接把人类关节角当机器人命令。[项目页](https://humanoid-manipulation-interface.github.io/)
- **HoMMI**：跨具身 hand-eye policy 先用具身无关视觉表示和 relaxed head action 缩小差异，WBC 再在机器人物理约束下实现 hand-eye trajectories。原文明确写 “realizes hand-eye trajectories through coordinated whole-body motion”。[论文](https://arxiv.org/abs/2603.03243)
- **Interactive Navigation**：对象未知动力学先通过交互学习为 SE(2) 表示，再进入 MPPI；这是“模型学习服务于规划”，不是上身-底盘 WBC。[论文](https://arxiv.org/abs/2410.13418)
- **SPARK**：将 nominal policy 与 safe controller 解耦，算法模块可组合；上游若输出 EE/关节/速度参考，SPARK 只在安全约束激活时修改它。[论文](https://arxiv.org/abs/2502.03132)
- **Topology-Driven**：全局层生成不同同伦类 guidance trajectories，多个局部规划器并行优化，滚动时域选择最低代价可行轨迹。[论文](https://arxiv.org/abs/2401.06021)

## 7. 维度 B：环境感知与避障如何进入系统

下面按“感知到的信息有没有真正进入避障/安全约束”分类。RGB-D、相机或点云存在，并不自动意味着系统进行了环境避障。

| B 类 | 感知与避障机制 | 对应论文 | 避障覆盖范围 |
|---|---|---|---|
| B1 | **预测/几何显式进入全身约束优化** | 1 Keep it Upright；4 REMANI；15 Articulated Object Interaction | 覆盖全身或机器人-障碍物碰撞；可处理静态，部分可处理动态障碍。 |
| B2 | **动态对象感知驱动闭环操作/移动** | 2 Interactive Navigation；11 On-The-Move | 目标/可推动物体随观测更新；重点是交互与抓取时机，不等价于全身碰撞规避。 |
| B3 | **多候选/拓扑式底盘动态避障** | 13 Topology-Driven | 纯轮式底盘在人群中的绕行；显式维护多个同伦类，避免局部最优抖动。 |
| B4 | **安全过滤与外部感知解耦** | 14 SPARK | 障碍物/人由 AVP、动捕或其他外部传感器提供；安全层用安全指数/CBF 类方法过滤名义控制。 |
| B5 | **感知服务于任务定位/泛化，但未实现外部避障** | 3 Holistic；9 HoMeR；10 HERO；12 HoMMI；7 HuMI | 可使用视觉、点云、VLM 或头戴相机定位/主动感知；不应将其宣传为环境碰撞规避。 |
| B6 | **地形/本体鲁棒性，而非外界障碍物避让** | 5 Whole-body EE Pose Tracking；6 UMI on Legs；8 HiWET | 处理地形、基座扰动、行走稳定性和漂移；未提供显式障碍物地图/碰撞约束。 |

### B1. 显式碰撞约束进入全身规划

- **Keep it Upright**：用障碍物和机器人碰撞球之间的距离向量 `d(x)`，并要求其非负；实时观测到抛射物后 MPC 更新，实验证明可在平衡瓶子的同时避开来球。原文定义 `d(x)` 为“distances between all pairs of collision spheres representing obstacles and the robot body”。[论文 PDF](https://www.dynsyslab.org/wp-content/papercite-data/pdf/heins-ral23.pdf)
- **REMANI-Planner**：环境自适应搜索根据环境复杂度调整搜索维度，随后时空优化显式考虑 whole-body safety。感知环境先影响路径拓扑/维度选择，再影响连续轨迹优化。[项目页](https://robotics-star.com/REMANI-Planner/)
- **Articulated Object Interaction**：RGB-D 支撑 object-centric 操作计划，agent-centric 最优控制负责在 moving obstacles 中安全跟踪。重点在“物体操作约束”和“动态环境中的全身安全”分层处理。[论文](https://arxiv.org/abs/2103.10534)

### B2--B4. 动态交互、底盘避障和安全过滤

- **Interactive Navigation**：视觉不足以确定物体动力学时，机器人通过推/交互形成自适应 SE(2) 动力学模型，MPPI 用它判断推开、绕开或继续前进。原文的关键词是 “handling objects with unknown dynamics” 和 “guide the robot's interactions”。[论文](https://arxiv.org/abs/2410.13418)
- **On-The-Move**：感知的 target pose 实时进入底盘控制器、reacher 和 final-phase visual servo；它能抓取不可预测运动物体，但论文的主目标是反应式抓取，不是障碍物地图避障。[论文](https://arxiv.org/abs/2212.06991)
- **Topology-Driven**：动态障碍导致非凸自由空间时，全局层保留多条不同同伦类绕行策略，局部层并行优化，避免单一优化器落到糟糕的局部极小值。[论文](https://arxiv.org/abs/2401.06021)
- **SPARK**：外部感知给出障碍物状态，安全控制器按照预设安全距离在接触前修正控制；其论文定位为可配置安全 benchmark/toolbox，而非新的环境语义感知器。[项目页](https://intelligent-control-lab.github.io/spark/)

### B5--B6. 必须避免的误读

- **HoMeR** 使用第三视角点云、腕部 RGB-D、可选 VLM 关键点，并在 WBC 中做**自碰撞**约束；论文也明确承认 “does not consider collisions with the external environment”。因此它是精细视觉操作，不是外部环境避障方案。[论文 PDF](https://homer-manip.github.io/assets/paper.pdf)
- **Holistic** 有闭环视觉抓取能力和关节限制/可操纵性约束，但没有环境障碍物规划模块；将其归为 B5 而非 B1。
- **HERO** 的开放词汇视觉用于识别和重定位目标，利用残差前向模型修正 EE 跟踪；公开方法描述没有给出全身外部障碍距离约束或动态避障器。[项目页](https://hero-humanoid.github.io/)
- **HuMI/HoMMI** 的视觉重点是示范采集、全局上下文和主动感知。特别是 HoMMI 以头戴相机解决“看哪里”，不等于建立障碍物避让闭环。[项目页](https://hommi-robot.github.io/)
- **Whole-body EE Pose Tracking、UMI on Legs、HiWET** 处理的是崎岖地形、基座扰动、稳定性和世界系漂移。它们展示高动态全身协调，但原文公开目标中没有环境几何避障模块，不能替代 B1/B3/B4。

## 8. 面向轮式高动态高精度操作的组合建议

若目标是“轮式底盘和上身整体参与、高动态、末端高精度、同时可避障”，题单中没有单篇完整覆盖所有层次，较合理的系统组合是：

1. **任务接口与执行**：采用 HoMeR 的“绝对 EE keypose + 局部 EE delta”接口，或 Keep it Upright 的任务空间约束 MPC；底层联合求解 base+arm。
2. **全身几何安全**：采用 Keep it Upright/REMANI 的机器人全身距离约束和动态重规划，而不是只在末端做视觉伺服。
3. **人机共融动态安全**：在名义 WBC/MPC 外再叠加 SPARK 一类安全过滤器，接收外部障碍物状态；其作用是兜底而不是取代规划。
4. **局部精度**：近接触阶段切换到 HoMeR 的 delta action 或 On-The-Move 的 EE velocity visual servo；此时保持底盘仍参与 WBC，避免“先停底盘后操作”的解耦。

这四层分别对应任务语义、全身可行性、动态安全和精细闭环；把它们混为单一“避障模块”会丢失必要的系统职责边界。
