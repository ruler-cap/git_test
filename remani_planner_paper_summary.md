# REMANI-Planner 论文解读

**论文**：*Real-time Whole-body Motion Planning for Mobile Manipulators Using Environment-adaptive Search and Spatial-temporal Optimization*  
**作者**：Chengkai Wu, Ruilin Wang, Mianzhi Song, Fei Gao, Jie Mei, Boyu Zhou  
**发表**：IEEE ICRA 2024，pp. 1369-1375，DOI: [10.1109/ICRA57147.2024.10610192](https://doi.org/10.1109/ICRA57147.2024.10610192)  
**项目/代码**：[REMANI-Planner 项目页](https://robotics-star.com/REMANI-Planner/)；[GitHub](https://github.com/Robotics-STAR-Lab/REMANI-Planner)，GPL-3.0。

## 一句话结论

这是一套面向**差速轮式移动操作臂**的实时全身运动规划器。它让底盘和机械臂共同生成无碰、平滑且满足速度/加速度约束的轨迹，并在狭窄空间中保持末端任务位姿约束。其强项是实时全身路径与轨迹规划，不是力矩级 WBC、接触力控制或物体动力学控制。

## 具体任务

### 1. 密集三维环境中的移动操作

机器人从当前全身状态移动到目标全身状态，需要在桌下穿行、绕开地面及空中障碍、通过狭窄通道，同时让底盘和机械臂都参与避障。实机在 7 m x 5 m 的 craft room 中验证，环境包括桌子、货架和方形障碍物。

### 2. 真实推门任务

末端接触门后，机器人边移动边推门。任务约束要求末端：

- 在位置上贴合目标圆弧轨迹，容差 `d_p = 0.05 m`。
- 姿态相对门面保持近似垂直，容差 `d_o = 0.15 rad`。
- 推门过程中，初始的末端位姿误差会收敛并维持在上述容差内。

这说明其“末端精度”来自显式的末端位置/姿态约束，而不是通过视觉伺服或力传感闭环实现。

## 系统建模

平台由**差速驱动底盘 + L 自由度机械臂**组成，控制输入是左右轮角速度和各关节角速度：

`u = [omega_l, omega_r, q_dot_m]`

全身规划状态为：

`q = [q_x, q_y, psi, q_1, ..., q_L]`

其中 `(q_x, q_y, psi)` 是底盘平面位姿。论文将整机近似为覆盖底盘和每个机械臂连杆的碰撞球集合，以避免用一个过大的整体包络造成保守避障。

## 核心方法

整体是两阶段流水线：

`环境自适应路径搜索 -> 路径作为初值 -> 时空全身轨迹优化 -> 速度控制指令`

### 阶段一：环境自适应路径搜索

目标是在保证可达性的前提下降低高维搜索负担。

1. **Hybrid A* 规划底盘路径**
   - 先在三维底盘状态 `(x, y, yaw)` 中生成平滑、安全、满足差速车运动学的低代价路径。
   - 路径被拆成前进或倒车的多个子段，并附带时间分配。

2. **多层约束 RRT*-Connect 规划机械臂路径**
   - 对底盘路径的每个 waypoint，构造一个对应的机械臂可行构型层。
   - 每一层中的机械臂关节状态与该底盘 waypoint 组合后必须无碰并满足约束。
   - 搜索树只能在相邻层之间扩展，因此既保留底盘 waypoint 的时序关系，又避免直接在整个 `(3+L)` 维空间全局采样。
   - 该步骤使用双向树、`SAMPLE -> NEAREST -> STEER -> CONNECT` 以及相邻层 rewire。

3. **狭窄区域局部全身搜索回退**
   - 若“先底盘、后机械臂”的降维搜索在窄门、低矮桥洞等区域断开，算法定位中断段。
   - 仅对该段在完整 `(3+L)` 维构型空间执行多起点、多终点 RRT*-Connect，而不是从头全局重规划。
   - 这使其在普通环境中快，在强耦合狭窄区域仍具完整性。

### 阶段二：空间-时间全身轨迹优化

将第一阶段的离散无碰路径变成可执行的连续轨迹。

1. **MINCO 参数化**
   - 每个路径段表示为 `(3+L)` 维分段多项式；同时优化多项式参数和每段执行时间。
   - MINCO 使优化变量数量不随约束采样点数量线性增长，适合高维实时优化。

2. **目标函数**
   - 最小化高阶导数控制代价，获得平滑运动。
   - 最小化总执行时间，鼓励敏捷轨迹。
   - 对各类不等式约束施加罚函数；对非线性强的位置加密采样，改善收敛。

3. **约束转录**
   - 将原始约束优化改写为带罚项的无约束优化问题，再用 MINCO 求解。
   - 这是其保持计算规模可控的关键，不是传统逐点、全部显式约束的直接非线性规划。

## 约束设计

| 类别 | 具体约束 | 作用 |
| --- | --- | --- |
| 环境碰撞 | ESDF 距离场中每个碰撞球中心到障碍的距离必须大于球半径 | 底盘、连杆共同避开三维障碍。 |
| 自碰撞 | 任意两条连杆碰撞球的中心距离大于半径和 | 防止机械臂折叠或底盘-上身干涉。 |
| 轮子可行性 | 左右轮角速度、角加速度上限；最小底盘速度 | 将差速底盘非完整运动学写入轨迹约束。 |
| 关节可行性 | 各关节位置、速度、加速度上限 | 输出可跟踪的上身轨迹。 |
| 末端任务 | 末端位置距离与姿态角误差均受容差限制 | 支持推门、末端沿指定轨迹运动等任务。 |

## 实验结果

| 测试 | 结果 |
| --- | --- |
| 局部全身搜索消融 | 桥洞高度从 0.7 m 降至 0.4 m 时，缺少局部全身搜索的基线成功率从 98.16% 降至 0%；本文方法均为 100%。 |
| 与 RRT*-Connect 的路径搜索比较 | Cuboids 场景：57.22 ms、100% 成功率；RRT*-Connect：119.67 ms、43.50%。Bridge 场景：12.96 ms、100%；RRT*-Connect：107.37 ms、12.62%。 |
| 与 CHOMP 的轨迹优化比较 | Cuboids：294.17 ms、92.35% 成功且动力学可行；CHOMP：5791.48 ms、68.20%、动力学不可行。Bridge：175.67 ms、100%；CHOMP：2279.35 ms、98.79%、动力学不可行。 |
| 实机端到端规划 | 论文图示任务的机载规划时间为 419.2 ms；文中还报告另一实例为 302.9 ms。 |

## 与你关注方向的关系

### 适合直接借鉴的部分

- 底盘和上身不会被固定为两个串行阶段：最终轨迹在完整 `(3+L)` 状态空间内联合优化。
- 适合高自由度轮式移动操作臂在密集环境中进行敏捷避障、穿越窄通道和保持末端姿态。
- 其“局部切换到全维搜索”的策略很适合工程实现：正常场景保持快，只有真正强耦合的局部区域才付出高维搜索代价。
- 碰撞球 + ESDF + MINCO 是可移植性较强的实现组合，仓库已提供 FastArmer 和 UR5 配置示例。

### 不能直接解决的部分

- 没有像 `Keep it Upright` 那样建模托盘物体、摩擦锥、滑移/倾覆或载荷惯性。
- 不做力矩级全身动力学控制；差速底盘和机械臂均以速度接口执行。
- 推门只约束末端位姿，未建模接触力、门铰链动力学或未知环境刚度。
- 对动态障碍的核心论文实验重点是静态稠密点云环境；若用于人员/飞来物，需要增加动态预测与更高频的安全过滤层。

## 建议的系统定位

若要构建“高动态轮式底盘 + 上身一体 + 高精度末端”的系统，可以将 REMANI-Planner 放在**全身轨迹生成层**：它输出带时间参数、满足自碰撞和运动学边界的全身参考。其下可接全身 MPC/阻抗控制跟踪，末端接触任务则再加入力/导纳闭环；托盘运输还需把 `Keep it Upright` 的物体稳定约束并入轨迹优化或 MPC。
